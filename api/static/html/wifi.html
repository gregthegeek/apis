WiFi API

The WiFi API provides information about WiFi usage around campus. This is particularly useful when determining how many people are at a certain place on campus. The only locations currently supported are: ratty, vdub, jos, andrews, and blueroom. More locations may be added in the future. Check the /wifi/locations endpoint (documented below) for the most up-to-date information.

Count Users at a Location

  • Endpoint: /wifi/count
  • Arguments:
    • location <string>: one of the location names mentioned above
    • history <boolean> (optional): if true, measurements from the past hour will be returned
  • Returns: A JSON dictionary of the form: 
        {'location': STRING, 
     'timestamp': FLOAT (Unix Timestamp),
     'count': INTEGER }
    Or, if the history argument is true, the response will contain multiple timestamps and counts: 
        {'location': STRING, 
     'history': [{'timestamp': FLOAT, 'count': INTEGER},
                 {'timestamp': FLOAT, 'count': INTEGER},
                 ...]}

Get All Location Names

  • Endpoint: /wifi/locations
  • Arguments: None
  • Returns: A list of location names in the form: 
        {'locations': [STRING, STRING, ...]}
  • Note: Each string is the name of a location.

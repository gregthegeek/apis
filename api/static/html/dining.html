Dining API

The Dining API provides information about dining hall hours and menus. The eatery names are ratty and vdub. Currently, only the Ratty and VDub are supported.

Note: All arguments are required unless denoted otherwise.

Get Menus by Date

  • Endpoint: /dining/menu
  • Arguments:
    • eatery <string>: either 'ratty' or 'vdub'
    • year <integer> (optional): defaults to current year
    • month <integer> (optional): defaults to current month
    • day <integer> (optional): defaults to current day
    • hour <integer> (optional): defaults to current hour
    • minute <integer> (optional): defaults to current minute
  • Returns: The list of menus satisfying the given arguments and their count in the form: 
        {'num_results': INTEGER, 
     'menus': [{'eatery': STRING, 
               'year': INTEGER, 
               'month': INTEGER, 
               'day': INTEGER, 
               'start_hour': INTEGER, 
               'start_minute': INTEGER, 
               'end_hour': INTEGER, 
               'end_minute': INTEGER, 
               menu_section1: [STRING, STRING, ...],
               menu_section2: [STRING, STRING, ...],
               ...},
              ...]}
  • Note: Each eatery has different menu sections (e.g. "Bistro", "Main Menu", etc) each of which contains a list of foods in that section.

Get Menus by Food

  • Endpoint: /dining/find
  • Arguments:
    • food <string>: name of the food item (e.g. "pulled%pork%20sandwich")
  • Returns: A list of menus containing the given food in the form: 
        {'food': STRING, 
     'menus': [menu1, menu2, ...]}
  • Note: Each menu is a dictionary laid out exactly like in the response to "/dining/menu" requests.
  • Note: The given food may be autocorrected to match a similar food.

Get Hours

  • Endpoint: /dining/hours
  • Arguments:
    • eatery <string>: either 'ratty' or 'vdub'
    • year <integer> (optional): defaults to current year
    • month <integer> (optional): defaults to current month
    • day <integer> (optional): defaults to current day
  • Returns: The list of hours satisfying the given arguments and their count in the form: 
        {'num_results': INTEGER, 
     'hours': [{'eatery': STRING, 
               'year': INTEGER, 
               'month': INTEGER, 
               'day': INTEGER, 
               'open_hour': INTEGER, 
               'open_minute': INTEGER, 
               'close_hour': INTEGER, 
               'close_minute': INTEGER }
              ...]}
  • Note: Because they close/reopen throughout the day, some eateries may have multiple results per day.

Find Open Eateries

  • Endpoint: /dining/open
  • Arguments:
    • year <integer> (optional): defaults to current year
    • month <integer> (optional): defaults to current month
    • day <integer> (optional): defaults to current day
    • hour <integer> (optional): defaults to current hour
    • minute <integer> (optional): defaults to current minute
  • Returns: A list of hours objects, each of which is the same as a response from /dining/hours requests. 
        {'open_eateries': [hours_1, hours_2, ...]}

Find All Food Ever Served at an Eatery

  • Endpoint: /dining/all_food
  • Arguments:
    • eatery <string>: the eatery to get all food ever served at
  • Returns: A list of all the food ever served at that eatery. 
        {'eatery': STRING,
     'food': [STRING, STRING, STRING, ...]}
  • Note: This is especially useful for searching for specific food names with the /dining/find endpoint.

Getting Started

To access Brown's data, you'll need a Client ID. Your Client ID will be included with every request you make to the API. It's sort of like your secret key to all of the data we provide (so don't share it with other developers!).

Get a Client ID

Once you have your Client ID, you can begin requesting data from our various APIs. Our base URL is "api.students.brown.edu". Every request will require your Client ID (and possibly other information). For instance, the following URL will return a JSON object representing the current number of people in the Ratty:


https://api.students.brown.edu/wifi/count?client_id=YOUR-CLIENT-ID&location=ratty

These requests and all their different arguments can become very confusing. That's why we provide detailed documentation for you to reference. Some APIs also have language-specific libraries to make it even easier to make requests. For example, the Dining API has a Python library that helps simplify API requests. You can find documentation and a list of all these libraries on our resources page.
